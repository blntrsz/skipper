# Implement `GitHubProjectService`

## Goal

- Replace stub logic in `packages/core/src/workspace/adapter/github-project.service.ts` with real `clone` + `branch` behavior, reusing existing core git/path/fs patterns.

## Critical files

- `packages/core/src/workspace/adapter/github-project.service.ts`
- `packages/core/src/workspace/port/project.service.ts`
- `packages/core/src/workspace/domain/project.model.ts`
- `packages/core/src/internal/Git/GitService.ts`
- `packages/core/src/internal/Git/ShellGitService.ts`
- `packages/core/src/Sandbox/adapter/WorkTreeService.ts`
- `packages/core/src/domain/Path.ts`
- `packages/core/src/internal/Shell/ShellService.ts`
- `packages/cli/src/Sandbox.ts`

## Recommended approach

1. Keep changes minimal and local to `packages/core/src/workspace/adapter/github-project.service.ts` unless wiring/tests force one extra file.
2. In `clone(project)`:
   - Build repo slug as `${project.namespace}/${project.name}`.
   - Use `Path.repositoryRoot()` + `Path.makeRepositoryPath(project.name)` for local paths.
   - Use `FileSystem.FileSystem` to create repo root and no-op if repo already exists.
   - Reuse shell service to check `gh` is installed.
   - Run `gh repo clone` from repo root, preferably with explicit target dir `project.name`.
   - Map shell/fs failures to `ProjectError` with repo-specific messages.
3. In `branch(project)`:
   - Treat `main` as no-op.
   - Reuse `GitService.ensureRepositoryExists(project.name)` before creating worktree.
   - Build canonical worktree paths with `Path.makeWorkTreeRepositoryPath` and `Path.makeWorkTreePath`.
   - Use `FileSystem.FileSystem` to no-op if worktree already exists, else create parent dir.
   - Reuse `GitService.createWorkTree(...)` with `{ repository: project.name, branch: project.branch }`.
   - Map git/fs failures to `ProjectError`.
4. Keep logic thin: compose existing `Path`, `GitService`, `ShellService`, and `FileSystem` services; do not reimplement git/worktree rules inline.
5. If current layer lacks needed deps, add only the minimal imports/wiring in the same file.

## Behavior choices

- `clone`: idempotent if local repo dir already exists.
- `branch`: local worktree creation only; no remote GitHub branch creation.
- `branch`: fail if repo is not cloned locally.
- `main`: no worktree create.
