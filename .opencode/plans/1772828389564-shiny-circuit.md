# Worktree root check

## Goal
- Ensure non-main worktree create succeeds when `~/.local/share/skipper/worktree` or `~/.local/share/skipper/worktree/<repo>` does not exist.

## Recommended changes
- Update `src/Sandbox/adapter/TmuxWorkTreeService.ts`.
- In non-main create flow, compute `const workTreeRepositoryPath = WorkTreePath.makeRepositoryPath(config)`.
- Before `git.createWorkTree(repositoryPath, workTreePath)`, call `fs.makeDirectory(workTreeRepositoryPath, { recursive: true })`.
- Remove the current `fs.makeDirectory(repositoryPath, { recursive: true })` call; it creates the repo checkout path, not the worktree parent.
- Keep `src/internal/GitService.ts` unchanged; FS prep belongs in sandbox adapter.

## Branch lookup hardening
- Also update `src/Sandbox/Service.ts`.
- In branch-omitted flow, check whether `WorkTreePath.makeRepositoryPath({ repository, branch: "main" })` exists before fuzzy search.
- If it exists, keep current fuzzy behavior with `additionalOptions: ["main"]`.
- If it does not exist, skip directory search and default branch to `"main"`.
- This keeps first-worktree UX stable on fresh machines and avoids depending on fuzzy search against a missing path.

- Re-run same create; confirm reuse path still works.
- Run create without `--branch` when no worktree dir exists; confirm default/fuzzy flow still resolves and no crash.
