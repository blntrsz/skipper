# Refactor Session Module to Hexagonal Architecture

## Overview

Refactor `packages/cli/src/command/session/` and create `packages/core/src/session/` module following the hexagonal architecture pattern established in `packages/core/src/workspace/` and `packages/core/src/task/`.

## Current Structure

### CLI (`packages/cli/src/`)

- `Session.ts` - Single file with all 7 commands (create, list, get, update-state, add-message, messages, delete)
- `command/session/index.ts` - Imports from Session.ts

### Core (`packages/core/src/`)

- `domain/Session.ts` - Session model
- `domain/SessionMessage.ts` - SessionMessage model
- `Session/SessionService.ts` - Service interface
- `Session/Repository.ts` - SQL repository (no port abstraction)
- `Session/SqlSessionService.ts` - SQL service implementation

## Implementation Steps

### 1. Core - Create session/domain/

**Files to create:**

- `session/domain/session.model.ts` - Move from `domain/Session.ts`, rename exports
- `session/domain/session-message.model.ts` - Move from `domain/SessionMessage.ts`
- `session/domain/index.ts` - Export models

**Changes:**

- Rename `Session` class to `SessionModel` (following ProjectModel pattern)
- Rename `SessionMessage` to `SessionMessageModel`
- Keep all schema definitions unchanged

### 2. Core - Create session/port/

**Files to create:**

- `session/port/session.repository.ts` - Extract repository interface from Repository.ts
- `session/port/session.service.ts` - Move from SessionService.ts
- `session/port/index.ts` - Export service and repository

**Pattern from TaskRepository:**

```typescript
export interface SessionRepository {
  insertVoid: (session: SessionModel) => Effect.Effect<void, ...>;
  findById: (id: SessionId) => Effect.Effect<SessionModel, ...>;
  list: () => Effect.Effect<ReadonlyArray<SessionModel>, ...>;
  listMessages: (sessionId: SessionId) => Effect.Effect<...>;
  insertMessage: (message: SessionMessageModel) => Effect.Effect<void, ...>;
  delete: (id: SessionId) => Effect.Effect<void, ...>;
  deleteMessagesBySessionId: (id: SessionId) => Effect.Effect<void, ...>;
}
export const SessionRepository = ServiceMap.Service<SessionRepository>("SessionRepository");
```

### 3. Core - Create session/adapter/

**Files to create:**

- `session/adapter/sql-session.repository.ts` - Rename from Repository.ts
- `session/adapter/sql-session.service.ts` - Rename from SqlSessionService.ts
- `session/adapter/index.ts` - Export layers

**Changes:**

- Update imports to use new domain models
- Ensure layers provide to port interfaces, not inline types

### 4. Core - Create session/use-case/

**Files to create:** (one file per operation, following task pattern)

- `create-session.use-case.ts`
- `list-sessions.use-case.ts`
- `get-session.use-case.ts`
- `update-session-state.use-case.ts`
- `add-session-message.use-case.ts`
- `list-session-messages.use-case.ts`
- `delete-session.use-case.ts`
- `index.ts`

**Pattern:**

```typescript
export const createSession = Effect.fn("session.create")(function* (data: SessionCreate) {
  const service = yield* SessionService;
  return yield* service.create(data);
});
```

### 5. Core - Create session/index.ts

```typescript
export * from "./domain";
export * from "./port";
export * from "./adapter";
export * from "./use-case";
```

### 6. Core - Create runtime layer

**File:** `packages/core/src/runtime/local-session.runtime.ts`

```typescript
import { Layer } from "effect";
import { SqlSessionServiceLayer } from "../session/adapter/sql-session.service";
import { SqlSessionRepositoryLayer } from "../session/adapter/sql-session.repository";
import { makeDatabaseLive } from "../internal/DatabaseService";

const sessionLayerWithRepo = Layer.provide(SqlSessionServiceLayer, SqlSessionRepositoryLayer);
export const localSessionLayer = Layer.provide(sessionLayerWithRepo, makeDatabaseLive());
```

### 7. Core - Update exports

**Update packages/core/src/index.ts:**

- Remove old exports: `Session`, `SessionMessage`, `SessionService`
- Add new export: `export * as Session from "./session"`

**Update packages/core/src/runtime/index.ts:**

- Add `export * from "./local-session.runtime"`

### 8. CLI - Refactor command files

**Create session.common.ts:**

```typescript
import { Session } from "@skippercorp/core/session/domain";
import { Effect } from "effect";
import { Flag } from "effect/unstable/cli";

export const idFlag = Flag.string("id").pipe(
  Flag.withSchema(Session.SessionId),
  Flag.withDescription("Session id"),
);

export const printJson = (value: unknown) =>
  Effect.sync(() => {
    process.stdout.write(`${JSON.stringify(value, null, 2)}\n`);
  });
```

**Create one file per command:**

- Each file imports use-case from `@skippercorp/core/session/use-case`
- Each file pipes effect through `Effect.provide(localSessionLayer)`
- Follow task command files as template

### 9. CLI - Update index.ts

**Update packages/cli/src/command/session/index.ts:**

- Import all command files
- Export SessionCli with all subcommands

### 10. CLI - Delete old Session.ts

Remove `packages/cli/src/Session.ts`
