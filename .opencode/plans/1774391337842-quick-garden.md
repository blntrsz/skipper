# Clone Command Plan

## Goal

- Add `sk clone git@github.com:owner/repo.git`.
- Clone into `~/.local/share/github/<repo>` on the repo default branch.
- Stop after clone. No tmux attach. No worktree branch setup.

## Recommended Approach

- Add `packages/cli/src/command/clone.command.ts`.
  - Define a top-level `clone` command with required positional `remote` via `Argument.string("remote")`.
  - Parse `git@github.com:owner/repo.git` into `{ namespace, name }`.
  - Validate host + shape early; reject malformed input, extra path segments, missing `.git`, non-GitHub remotes.
  - Build `new Workspace.ProjectModel({ namespace: owner, name: repo })`.
  - Call `Workspace.initWorkspace(project)`.
- Update `packages/cli/src/command.ts` to register `cloneCommand` in `rootCommand`.
- Do not reuse `pickProject` from `packages/cli/src/command/workspace/workspace.common.ts`; it is interactive and repo-name oriented.
- Update `README.md` examples to use the SSH remote input.
- Reuse core clone logic unchanged:
  - `packages/core/src/workspace/use-case/init-workspace.use-case.ts`
  - `packages/core/src/workspace/adapter/work-tree-sandbox.service.ts`
  - With `branch === undefined`, current flow already clones only the main checkout; plain `git clone` lands on remote HEAD/default branch.

## Tests

- Add CLI parse test for valid `git@github.com:owner/repo.git` -> `namespace` + `name`.
- Add validation tests for malformed remotes.
- Add dry-run coverage for `sk clone git@github.com:owner/repo.git --dry-run`.
- Assert dry-run emits only `git clone git@github.com:owner/repo.git ...`.
- Assert no `tmux` commands and no `git worktree add` for clone-only flow.
- If easy from current seams, add idempotency coverage for already-existing main repo path.

## Edge Cases

- Existing local repo path should no-op, not reclone.
- Owner/name collisions remain with current storage model: `foo/api` and `bar/api` both map to `~/.local/share/github/api`.
- Support only GitHub SSH remotes in first pass; reject HTTPS and other hosts.
