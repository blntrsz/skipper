# Migration Implementation: Switch to Record-Based Migrations

## Overview

Replace broken filesystem-based migrations with `SqliteMigrator.fromRecord` using static imports.

## Approach

- Keep migrations in separate files (current pattern)
- Use 4-digit naming: `0001_create_tasks`
- Create a registry file that imports and exposes all migrations
- Remove filesystem dependency from migrator

## Implementation

### 1. Create Migration Registry

**File:** `src/migrations/index.ts` (NEW)

```typescript
import { Effect } from "effect";
import { SqlClient } from "effect/unstable/sql";
import createTasks from "./0001_create_tasks";

export const migrations = {
  "0001_create_tasks": createTasks,
};
```

### 2. Update DatabaseService

**File:** `src/internal/DatabaseService.ts`

**Changes:**
- Import `migrations` from `"../migrations"`
- Replace: `SqliteMigrator.fromFileSystem(MIGRATIONS_DIR)` → `SqliteMigrator.fromRecord(migrations)`
- Remove: `schemaDirectory` option
- Remove: `BunFileSystem.layer` provider from `MigratorLive`
- Keep: `FileSystem` and `BunFileSystem` for `EnsureDatabaseDirectoryLive`

**Updated migrator config:**
```typescript
const migratorOptions = {
  loader: SqliteMigrator.fromRecord(migrations),
} as const;

const MigratorLive = SqliteMigrator.layer(migratorOptions);
// Remove: .pipe(Layer.provide(BunFileSystem.layer))
```

### 3. Future Migration Pattern

When adding new migrations:

1. Create file: `src/migrations/0002_description.ts`
```typescript
import { Effect } from "effect";
import { SqlClient } from "effect/unstable/sql";

export default Effect.gen(function* () {
  const sql = yield* SqlClient.SqlClient;
  yield* sql`ALTER TABLE tasks ADD COLUMN priority INTEGER DEFAULT 0`;
});
```

2. Update registry in `src/migrations/index.ts`:
```typescript
import createTasks from "./0001_create_tasks";
import addPriority from "./0002_description";

export const migrations = {
  "0001_create_tasks": createTasks,
  "0002_add_priority": addPriority,
};
```
