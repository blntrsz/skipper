# Plan

Implement `sandbox attach --create` as a compose of existing create + attach behavior.

## Files

- `packages/cli/src/command/sandbox/attach-sandbox.command.ts`
- `packages/cli/src/command/sandbox/sandbox.common.ts`
- `packages/cli/src/command/sandbox/create-sandbox.command.ts`

## Changes

1. Add a boolean `--create` flag to `attachSandboxCommand` only.
2. Refactor `pickProject(git, isNewBranch)` in `sandbox.common.ts` to a clearer mode-based API, eg `pickProject(git, { branchMode: "new" | "existing" })`.
3. Keep repo selection same as today: existing local repo autocomplete unless `--repository` is passed.
4. Make branch selection mode-specific:
   - `attach` -> existing branch picker
   - `attach --create` -> free-text new branch prompt
   - `create` -> free-text new branch prompt
5. Update `attachSandboxCommand` flow:
   - without `--create`: `pickProject(... existing ...)` then `Workspace.attachWorkspace(project)`
   - with `--create`: `pickProject(... new ...)` then `Workspace.initWorkspace(project)` then `Workspace.attachWorkspace(project)`
6. Update `createSandboxCommand` to use the refactored picker API with new-branch mode, behavior unchanged otherwise.

## Notes

- `Workspace.initWorkspace(...)` does not attach today, so `attach --create` must call both use cases.
- This also fixes current attach behavior: it now incorrectly uses the new-branch picker path.
- Fresh clone of a repo not already present locally may still depend on `ProjectModel.namespace`; current picker only returns local repos, so keep scope to existing local repos unless separate repo creation support is needed.
