Goal: add first-class repo clone support to `packages/core/src/workspace/`.

Current state

- `packages/core/src/workspace/**` does not clone repos today.
- Existing clone logic lives in `packages/cli/src/Sandbox.ts`: `gh repo clone <repo>` into `Path.repositoryRoot()`.
- Core git service only handles worktrees today: `packages/core/src/internal/Git/GitService.ts`, `packages/core/src/internal/Git/ShellGitService.ts`.

Recommended approach

1. Keep clone separate from `initWorkspace`.
   - `initWorkspace` should stay generic setup.
   - Add a dedicated `cloneWorkspace` use-case for repo acquisition.
2. Add a workspace repo port.
   - New file: `packages/core/src/workspace/port/repository.service.ts`.
   - Shape: `clone(input: { repository: string }): Effect<RepositoryPath, RepositoryError, ...>`.
   - Reason: clone is repo lifecycle, not raw fs or sandbox lifecycle.
3. Add a GitHub-backed adapter.
   - New file: `packages/core/src/workspace/adapter/github-repository.service.ts`.
   - Reuse current behavior: ensure `Path.repositoryRoot()` exists, then run `gh repo clone <repo>` there.
   - Use existing shell infra instead of calling Bun shell directly from use-case.
4. Add clone orchestration use-case.
   - New file: `packages/core/src/workspace/use-case/clone-workspace.use-case.ts`.
   - Flow: `fileSystem.init()` -> `repository.clone({ repository })`.
   - First pass: clone only. Do not auto-attach unless explicitly needed.
5. Fix workspace baseline gaps so module is coherent.
   - `packages/core/src/workspace/port/sandbox.service.ts`: add optional `attach` or remove mismatch.
   - `packages/core/src/workspace/use-case/destroy-workspace.service.ts`: fix bad import.
   - `packages/core/src/workspace/adapter/work-tree-sandbox.service.ts`: replace stub logging with minimal real impl or explicit no-op semantics.
6. Export + wire it.
   - Add `packages/core/src/workspace/index.ts`.
   - Export from `packages/core/src/index.ts`.
   - Wire layer(s) in `packages/core/src/Runtime.ts` and `packages/core/src/TestRuntime.ts`.
   - Optionally make `packages/cli/src/Sandbox.ts` `cloneCommand` call `cloneWorkspace` so clone logic lives in core once.
