# Plan: Port ShellTmuxService to workspace

## Goal

Port `@packages/core/src/internal/Tmux/ShellTmuxService.ts` to `@packages/core/src/workspace/` following the workspace's clean architecture pattern.

## Background

- `ShellTmuxService` implements `TmuxService` interface with `attachSession(sessionName, path)` method
- Uses tmux to create/attach sessions for sandbox workspace attachment
- Workspace uses Layer + ServiceMap pattern for dependency injection

## Steps

### 1. Create port interface

**File:** `packages/core/src/workspace/port/tmux.service.ts`

Create `TmuxService` port with:

- `TmuxError` tagged error class (mirroring ShellError pattern)
- `TmuxService` interface with `attachSession` method
- Error handling via Schema.TaggedErrorClass

### 2. Create adapter implementation

**File:** `packages/core/src/workspace/adapter/shell-tmux.service.ts`

Port `ShellTmuxService` logic using:

- `Layer.effect(TmuxService, Effect.gen(...))` pattern
- `Effect.fn()` wrappers for methods
- ShellService for shell operations

### 3. Export from workspace index

**File:** `packages/core/src/workspace/index.ts`

Add exports for new port and adapter

### 4. Integrate with SandboxService (optional future)

The `SandboxService.attach()` method could use TmuxService to attach to tmux sessions
