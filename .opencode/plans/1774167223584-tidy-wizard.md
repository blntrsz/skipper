# Plan: Migrate Task to Hexagonal Architecture

## Summary

Refactor Task from flat service pattern to hexagonal architecture (domain/port/adapter/use-case), matching the workspace pattern. Create CLI commands following sandbox pattern.

## Current State

- **CLI**: Single file `packages/cli/src/Task.ts` with all commands
- **Core**: Flat structure in `packages/core/src/Task/` (TaskService, SqlTaskService, Repository)
- **Domain model**: `packages/core/src/domain/Task.ts`

## Target State

Follow workspace pattern:

- **CLI**: `packages/cli/src/command/task/` with subdirectory structure
- **Core**: `packages/core/src/task/` with domain/port/adapter/use-case layers
